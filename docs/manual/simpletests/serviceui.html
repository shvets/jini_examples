Simple Service as UI Service

This example explains how to work with Service UI library.

This example has the following classes. The org.sf.jini.examples.serviceui.FilterFactory class is the UI Factory. Its main responsibility is to create UI frame (org.sf.jini.examples.serviceui.FilterFrame). This frame, in turn, holds a panel (org.sf.jini.examples.serviceui.FilterPanel) for displaying results. It takes our service as input parameter and performs method invocation for the service.

Another UI Factory class is org.sf.jini.examples.serviceui.FilterComponentFactory. It creates (org.sf.jini.examples.serviceui.FilterPanel) UI component.

We also have 2 classes - for client (org.sf.jini.examples.serviceui.FilterClient) and server (org.sf.jini.examples.serviceui.FilterService) sides.

We'll create 2 archives:
  • for service itself (target/lib-service/ui-filter-service.jar);
  • for client that wants to work with this service (target/docroot/service/ui-filter-service-dl.jar).

    • Now execute these commands in the following order:

    >start ant start.services
    >ant serviceui.service
    >ant serviceui.client

    In case of success, client will raise the frame window. If everything is ok, let's start the another example.
