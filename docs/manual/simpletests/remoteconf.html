Simple Remote Service over JERI started from configuration file

We'll develop a very simple service, based on the previous example. It is exactly the same (org.sf.jini.examples.remote.RemoteFilter, org.sf.jini.examples.remote.RemoteFilterImpl), except the service class. It should be aware of Jini lifecycle. It means that constructor method should be implemented in the following way: 
  public RemoteConfService(String args[], LifeCycle aLifecycle) throws Exception {
    // move code from main method here
  }
This example depends on the following services:
- reggie;
- http server for Jini libraries;
- http server for this service;

For this example these jar files will be created:
  • for service itself (target/lib/remoteconf-filter-service.jar);
  • for client that wants to work with this service (target/docroot/service/remoteconf-filter-service-dl.jar).


    • The service uses configuration file, located in src\main\config\remoteconf.config. Here where we define our transport protocol details.

    For starting this service, another configuration file is used: src\main\config\start-remoteconf.config).

    Run this commands:

    >start ant start.services
    >start ant remoteconf.service
    >ant remoteconf.client

    If you have some time before running the client, probably your new service has been expired. Restart your service and then run your client again.
