Administrable service

Let's try to create the service that supports remote adminstration.

This example is based on remoteservice primer. We will modify remote filter (org.sf.jini.examples.adm.AdmFilter) and it's implementation (org.sf.jini.examples.adm.AdmFilterImpl). The last class is implemented as administrable object. To support proxy object we use org.sf.jini.examples.common.ProxiedJiniService class.

The service (org.sf.jini.examples.adm.AdmFilterService) registers corresponding proxy object, whereas the client (org.sf.jini.examples.adm.AdmFilterClient) is looking for required service.

After the build we have 3 archives:
  • for service itself (target/lib/adm-filter-service.jar);
  • for client that wants to work with this service (target/docroot/service/adm-filter-service-dl.jar);
  • for activation framework (target/docroot/service/adm-filter-service-act.jar).


    • Now it's time to test them. Execute these commands in the following order:

    >start ant start.services
    >start ant phoenix.start
    >ant adm.service
    >ant adm.client

    The client will check the interfaces, implemented by proxy object and if it implements Administrable interface, the client will try to remove the service.

    To check all these events in dynamic manner, look at jini browser window. You will see how service is added to registrar and how it will be deleted. Don't forget to select the registrar from the Regisrar menu item.

    Afrer completing the tes, don't forget to shutdown Phoenix service:

    >start ant phoenix.stop
