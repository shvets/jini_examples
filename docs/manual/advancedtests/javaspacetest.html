JavaSpace test

In this example we make simple experiments with JavaSpace. Don't be afraid - it's just yet another Jini service (outrigger). Additional service that is required - the Transaction Manager (mahalo). Before example could be tested, the Transaction Manager should be registered and JavaSpace should be started. But don't worry, these services are already included:

>start ant start.services

Dependent services list:
- reggie;
- outrigger;
- mahalo;
- http server for Jini libraries;
- http server for this service;

The JavaSpace is the environment that could keep shared instances of the net.jini.core.entry.Entry type. We have one such representative: org.sf.jini.examples.javaspace.Message. The org.sf.jini.examples.javaspace.JavaSpaceTest class is the client, that looks for two services - JavaSpace and Transaction Manager. If these services are discovered, the client will try to create messages and write them ino JavaSpace without and with transaction support. The rest of the code will try to read/print/take these entries from the JavaSpace.

Now, compile example and test it:

>ant javaspace.test

You also could run another "hello-world-from-jini" example borrowed from blitz project. It's not dependent on "blitz" project though. In order to run this example, execute these commands:

>start ant blitzhelloworld.taker
>start ant blitzhelloworld.writer
