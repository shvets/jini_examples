Client listening for events

In this example we will create a client, registered for listening event from reggie service.

We have two additional classes here - for client (org.sf.jini.examples.eclient.EventFilterClient) and for service (org.sf.jini.examples.eclient.EventFilterService).

After the build we have 2 archives:
  • for service itself (lib-service/ec-filter-service.jar);
  • for client that wants to work with this service (docroot/service/ec-filter-service-dl.jar);

    • Now it's time to test them. Start this command:

    >start ant start.services

    For this example we have to take care of these services:
    - reggie;
    - http server for Jini libraries;
    - http server for this service;
    - http server for client code;

    Try to start the client:

    >start ant eclient.client

    You could see that the service could not be found. In this case the client will go to the waiting mode.

    Now register the service:

    >ant eclient.service

    After service registration look at client's console. Finally it got the event.
