Failover example

This example is based on similar example from IncaX company.

This example shows how to use a ProxyPreparer to detect (potential) failure of your service and then lookup a different instance of that service in the client's proxy. The ProxyPreparer then swaps the proxies so the client can continue using a different instance of the same service. We have implementation of this interface as org.sf.jini.examples.failover.FailoverPreparer.

How to run this example. First, start required Jini services:

>start ant start.services

Start one instances of the example service:

>start ant failover.service

Run the client example

>start ant failover.client

Start a second instance of the service;

>start ant failover.service

Stop the first of the ping service instances.

You client code should show that the proxy has switched services
