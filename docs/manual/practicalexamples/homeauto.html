Homeauto Services

In this example we'll make more experiments with FreeTTS library. The original source for this examples is located here.

Below we have 3 examples:
  • music service
  • time service
  • speech service
    •

    For all these examples we have the client (remote control). It is built with the help of ServiceUI library. It waits for services to be registered and when any service get registered, it adds this service to the client pane.

    The music service is mp3 player.

    The time service can speak current server side time. It also uses better tuned voice (only numbers and only in English language).

    The speech service can speak the entered text.

    First, you have to start standard Jini services and the client:

    >start ant start.services
    >start ant homeauto.client

    Now you can test examples.

    >start ant homeauto.music.service
    >start ant homeauto.time.service
    >start ant homeauto.speech.service

    After starting each service one more pane should be created on client side. By selecting one of the pane you can work with different services. For music service select the mp3 file and click "Play" button. For time service select the mp3 file and click "Play" button.
