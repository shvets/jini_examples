Test with NetLib library

In this example we'll make experiments with NetLib library. You should download the most resent version or use one that resides in the lib directory.

We have the following classes here:

- org.sf.jini.examples.netlib.NetlibClient - the test client;
- org.sf.jini.examples.netlib.NetlibService - the test service;
- org.sf.jini.examples.netlib.JiniServer - the class that represents Jini service as a server;
- org.sf.jini.examples.netlib.NetlibServer - the test server, extension of org.sf.jini.examples.netlib.JiniServer class.

Client listens for events. If the server is not started yet, the client will wait. After receiving the notification from the server, the client creates simple request and send it to the server. Server prepares responses and sends them to the client. Server class is just wrapper around service, so when we start the server, we register the service and when we stop the server, we terminate the service by cancelling the lease.

After the build we have 3 archives:
  • for the service itself (lib-service/netlib-service.jar);
  • for the client that wants to work with this service (docroot/service/netlib-service-dl.jar);
  • for the server that will notify the client about incoming events (docroot/client/netlib-client-dl.jar).


    • Ready? Execute these commands in the following order:

    >start ant netlib.service
    >ant netlib.client

    Well done.
